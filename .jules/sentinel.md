## 2025-05-14 - HTML Sanitization and Security Headers
**Vulnerability:** Widespread XSS vulnerabilities in multiple HTML pages where user-provided data from the API (donor names, hospital details, descriptions) was injected into the DOM using `innerHTML` without any sanitization. Also, missing security headers like CSP and X-Frame-Options.
**Learning:** In a legacy or simple static frontend architecture that relies heavily on manual DOM manipulation via `innerHTML`, the lack of a modern framework's automatic escaping makes XSS a systemic risk. Security headers were also entirely absent, leaving the application open to multiple common web attacks.
**Prevention:** Always use a central sanitization utility to escape data before injecting it into `innerHTML`. Prefer `.textContent` where possible as it natively handles escaping. Implement a robust Content Security Policy (CSP) and other security headers at the deployment level (e.g., `netlify.toml`) to provide multiple layers of defense.
